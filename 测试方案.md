# 验证微服务治理三大场景

# ========== 测试 1: Sentinel 降级 ==========
# 场景: 关停 product-service 后下单
# 步骤:
#   1. docker-compose up -d      # 启动所有服务
#   2. docker stop product-service  # 模拟商品服务宕机
#   3. 通过 Gateway 发起下单请求（POST /api/order）
# 预期结果:
#   - 返回 {"code": 503, "message": "商品服务繁忙，请稍后重试", "data": null}
#   - Sentinel Dashboard 显示降级触发
# 截图要点:
#   - Sentinel Dashboard 降级规则触发截图
#   - Postman/curl 响应截图

# ========== 测试 2: 幂等性验证 ==========
# 场景: 使用相同 orderToken 重复提交订单
# 步骤:
#   1. 获取一个新的 orderToken
#   2. 第一次提交 POST /api/order，携带该 token → 成功
#   3. 第二次提交相同 POST /api/order，携带相同 token → 失败
# 预期结果:
#   - 第一次返回 200，订单创建成功
#   - 第二次返回 400，"订单已提交或Token无效，请勿重复提交"
# 截图要点:
#   - 两次请求的请求体和响应体对比截图

# ========== 测试 3: Seata 分布式事务回滚 ==========
# 场景: 下单过程中，扣库存成功但插入订单失败
# 步骤:
#   1. 构造一个异常场景（如订单项包含不存在的 productId）
#   2. 发起下单请求
# 预期结果:
#   - 订单创建失败
#   - 已扣减的库存被自动回滚（数据库中库存数量恢复）
#   - Seata 日志中显示 rollback 操作
# 截图要点:
#   - Seata Dashboard 事务回滚记录截图
#   - 数据库库存前后对比截图
